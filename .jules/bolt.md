## 2024-11-20 - ItemStorage `get_item` N+1 bottleneck
**Learning:** In yagdb's `ItemStorage` architecture, calling `get_item(id)` performs a full object clone for Memory storage or a full disk deserialization/clone for Disk storage. Doing this in a tight loop like `node_matches` filtering or path traversal causes significant allocation overhead.
**Action:** Use the newly implemented `with_item(id, |item| { ... })` method instead. This method provides a zero-cost reference to the item via a closure, avoiding unnecessary memory allocations for read-only operations. When refactoring, retain `.unwrap()` to maintain the original failure mode.
