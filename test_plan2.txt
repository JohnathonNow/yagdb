Let's see where to inject the cancellation check.
The long-running loops in `execute` are:
1. Iterating over `plan.steps`.
2. Inside `execute_plan` where it does:
   - `for i in 0..in_res.rows`
   - `for node_id in nodes`
   - `for l_idx in 0..left_res.rows`
   - etc.

Actually, checking a thread-local variable or an `Arc<AtomicBool>` at key loop boundaries (e.g., at the start of `execute_plan` or at the start of the `for step in plan.steps` loop, or even in `find_nodes`, `match_edges_recursive`) is the most robust way.

Since `execute` is invoked from `handle_query` and `handle_query_stream` in `main.rs` and `server.rs`, we can detect client disconnect.
Wait, axum drops the future when the client disconnects!
If we spawn a `tokio::task::spawn_blocking`, we can pass a cancellation flag. When the axum handler future is dropped, a drop guard can set the flag to true. The `spawn_blocking` thread will see this flag and return early.

Let's design the changes:
1. Add `cancel_flag: Option<Arc<std::sync::atomic::AtomicBool>>` to `execute_plan` and `execute`? Or just `execute_with_cancel` which `execute` delegates to. But we need to pass it down.
Wait, instead of changing all `execute_plan` signatures (it has 6 args already), we can add `pub cancel_flag: Arc<std::sync::atomic::AtomicBool>` to `Graph` itself! But `Graph` is serialized via `bincode`.
We can use `#[serde(skip)]` and `#[serde(default = "default_cancel_flag")]` for deserialization.
Wait, `ItemStorage` doesn't need it. Just `Graph`.
```rust
    #[serde(skip)]
    #[serde(default = "default_cancel_flag")]
    pub cancel_flag: Arc<std::sync::atomic::AtomicBool>,
```
Wait, `Arc` does not implement `Default` unless `T` does. `AtomicBool::new(false)` is default. So `Arc::default()` is `Arc::new(AtomicBool::default())`, which works!

Then in `execute`, we can check:
```rust
if self.cancel_flag.load(std::sync::atomic::Ordering::Relaxed) {
    return Err("Query cancelled".to_string());
}
```
We should check this in:
- `execute`'s main step loop
- `execute_plan` (first line)
- `match_edges_recursive` (first line)
- `match_var_length_edges` (first line)

Then in `main.rs` `handle_query` and `handle_query_stream`, and in `raft/server.rs`:
